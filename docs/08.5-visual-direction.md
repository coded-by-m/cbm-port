# 08.5 — Visual Direction

# Coded by M — Direção Visual da Identidade Oficial

## Contexto

O Experience Lab cumpriu seu objetivo: validar a arquitetura técnica com cores
neutras para não contaminar decisões visuais durante testes.

Este documento define como a identidade visual oficial da Coded by M — já existente,
documentada no Brand Guide — deve ser aplicada ao site, aos componentes, ao 3D
e a cada camada da experiência.

A partir daqui, nenhum componente da Home deve ser construído sem esta referência.

---

## 1. Filosofia Visual

A identidade da Coded by M não é estética decorativa.

Ela é a externalização visual de um princípio de trabalho:

**Estrutura com propósito. Precisão sem excesso. Impacto controlado.**

O sistema visual tem três camadas de intenção:

**Fundo profundo** — o espaço onde as estruturas existem. Escuro, denso, mas não
vazio. O `#000F08` não é preto puro: tem uma temperatura verde quase imperceptível
que afasta o look genérico de dashboard escuro.

**Estrutura branca-quente** — linhas, nós, geometria, texto de corpo. O `#F5F2ED`
é off-white com temperatura quente, não fria. Jamais branco puro. Isso mantém o
visual premium e afasta a aparência cyberpunk.

**Sinal vermelho** — o único elemento que corta. O `#FB3640` existe como ruptura
deliberada. Não decora: sinaliza. Quando aparece, importa.

Esses três elementos — fundo profundo, estrutura quente, sinal vermelho — são a
gramática visual completa da Coded by M.

## 3. Papel de Cada Cor

### Deep Black `#000F08`

É o espaço. O universo onde tudo é construído.

Não é preto puro porque preto puro é sem temperatura. O `#000F08` tem uma presença
discreta de verde — a mesma temperatura do Forest Dark, porém extremamente densa.

Isso cria coerência entre fundo e card sem usar cinza neutro.

Uso: fundo de todos os Canvas 3D, fundo da página, fundo do Loading.

Nunca usar `#000000` nem `#0d0d0d` como fundo principal da Home.

---

### Signal Red `#FB3640`

É o único elemento que corta o espaço.

No logo, ele é a diagonal que atravessa as duas estruturas. Esta diagonal não é
ornamental — ela é a identidade do movimento: a marca corta, avança, define.

No site, o vermelho:
- Marca ações primárias (CTA principal)
- Identifica o elemento ativo (fragmento selecionado)
- Aparece como conector entre 3D e HTML overlay
- Sublinha títulos de seção com linha de `0.5px`
- Está sempre em baixa frequência — raridade é o que dá potência

Regra: o vermelho nunca aparece em área grande sem propósito de sinalização.
Uma borda de `1px`, uma linha de `0.5px`, um ponto — é suficiente.

---

### Off White `#F5F2ED`

É a estrutura. O que foi construído.

No logo, são as duas linhas estruturais do símbolo. No 3D, são os nós, as
arestas, o wireframe. No HTML, é o texto principal.

Jamais substituir por `#ffffff`. O branco puro elimina a temperatura quente
que define o caráter premium da marca.

Uso: texto de títulos e corpo, geometria 3D em estado padrão, elementos de
interface de alta importância.

---

### Forest Dark `#0E1810`

É a profundidade controlada. O espaço onde cards e overlays existem.

Tem a mesma temperatura do Deep Black, mas é ligeiramente mais claro — o
suficiente para criar diferenciação sem quebrar o ambiente escuro.

Uso: fundos de cards HTML, backgrounds de overlays, painéis de projeto, navbar
com transparência, separadores de seção.

---

## 4. Sistema de Iluminação

A Coded by M opera em **estúdio escuro com iluminação técnica**.

Não é sala iluminada. Não é neon genérico. Não é cyberpunk.

É o interior de uma estrutura — como olhar para dentro de um projeto antes de
ele ser finalizado.

### Regras de iluminação

**Fundo:** `#000F08` constante. Sem gradientes de fundo de página.

**Névoa 3D:** fog do Three.js com cor `#000F08`. Funde os elementos distantes
no fundo sem criar horizonte visível. Essencial para a sensação de profundidade
infinita.

**Brilho de estrutura:** os nós e arestas não emitem luz — têm cor sólida
`#F5F2ED` sobre o fundo escuro. O contraste cria a ilusão de brilho sem glow.

**Glow vermelho (uso restrito):**
Quando um elemento vermelho precisa de presença (fragmento ativo, CTA final),
pode receber um `radial-gradient` ou `box-shadow` vermelho com máximo de
`rgba(251,54,64,0.08)`. Nunca mais intenso.
Referência exata do Brand Guide: `rgba(251,54,64,0.06)`.

**Não usar:**
- `#0080ff`, azul neon, ou qualquer gradiente de cor fria
- `emissive` intenso em materiais 3D
- Bloom ou glow forte em post-processing
- Sombras coloridas que não sejam vermelho ou verde-escuro

## 5. Aplicação da Identidade no 3D

### Material dos nós (pontos)

```
cor: #F5F2ED  (MeshBasicMaterial — sem iluminação)
```

Estado ativo (hover/selecionado):
```
cor: #FB3640
```

Estado inativo/distante:
```
cor: #4A4844  (Gray 600 — recuo visual natural)
```

### Material das arestas (linhas)

Estado padrão:
```
cor: #F5F2ED  opacidade: 0.25 – 0.45 (varia por profundidade de camada)
```

Arestas mais distantes (camadas de fundo):
```
cor: #C8C4BE  opacidade: 0.12 – 0.20
```

Aresta de destaque (fragmento ativo, conector):
```
cor: #FB3640  opacidade: 0.9
```

### Terreno (TerrainMesh)

Camada frontal (mais próxima):
```
cor: #F5F2ED  opacidade: 0.35
```

Camada intermediária:
```
cor: #C8C4BE  opacidade: 0.18
```

Camada de fundo:
```
cor: #8A8780  opacidade: 0.08
```

Fog: cor `#000F08`, near e far ajustados para que a camada de fundo dissolva
gradualmente — nunca corte abrupto.

### Logo em 3D (CTAFormation)

O símbolo da Coded by M tem três paths:

```
Path 1 — Linha estrutural esquerda
  M11.5 148.039 V59.0391 L53.5 104.438
  cor: #F5F2ED  strokeWidth: 9  linecap: round  linejoin: round

Path 2 — Linha estrutural direita
  M130.5 103.039 V19.0391 L85.5 67.2944
  cor: #F5F2ED  strokeWidth: 9  linecap: round  linejoin: round

Path 3 — Diagonal vermelha (o sinal)
  M9.5 18.0391 L130.5 147.039
  cor: #FB3640  strokeWidth: 9  linecap: round
```

Animação de construção (CTAFormation):
1. Estrutura esquerda se desenha (Line2 com `dashOffset` decrescendo)
2. Estrutura direita se desenha
3. A diagonal vermelha corta de cima-esquerda para baixo-direita — mais rápido
   que as estruturas, como uma incisão

A diagonal vermelha sempre anima por último e mais rápido.
Ela nunca aparece primeiro.

## 6. Aplicação da Identidade nos Fragmentos

Os fragmentos representam projetos — estruturas que ainda estão sendo descobertas.

### Estado padrão (não descoberto)

```
faces: sem fill (wireframe)
arestas: #F5F2ED  opacidade: 0.3
nó do vértice: #F5F2ED  opacidade: 0.4  escala: 0.6
```

### Estado hover/ativo (descoberto)

```
arestas: #F5F2ED  opacidade: 0.8
nó central/ápice: #FB3640  escala: 1.2 (a "cabeça" vermelha que aponta o projeto)
escala geral do fragmento: 1.05 → 1.08 (suave)
```

### Marcador de hotspot

O ponto que sinaliza a existência de um projeto antes do hover:
```
cor: #FB3640  tamanho: pequeno (escala 0.3 – 0.4 relativo ao nó)
pulsação: sutil — amplitude 0.1, período 2.5s
```

O marcador não é decorativo. É o sinal vermelho que diz "aqui existe algo".

## 9. Aplicação da Identidade na Paisagem Digital

A Paisagem Digital é a principal assinatura visual. Ela deve parecer um
ambiente — não uma animação.

### Paleta 3D

Terreno base: Off White em opacidades decrescentes por profundidade
(`0.35` → `0.18` → `0.08`).

Névoa: Deep Black `#000F08`. A névoa funde as camadas de fundo no espaço.

Fragmentos padrão: mesma paleta do terreno.

Fragmento descoberto: aresta do ápice em `#F5F2ED` alta opacidade (`0.85`),
ponto de vértice em `#FB3640`.

```
background: #0E1810  border: 1px solid #1a2a1e

letter-spacing: 0.2em  text-transform: uppercase
```

Item ativo do filtro:
```

```

Item inativo:
```
color: #4A4844
```

## 10. Aplicação da Identidade no CTA Final

O CTA Final é o momento de convergência. O ambiente escuro persiste, mas o
símbolo se revela.

### Cena 3D

Fragmentos espalhados → convergem → formam o logo.

Durante a convergência:
- Fragmentos em `#F5F2ED` com opacidade crescente conforme aproximam
- Quando o logo se forma: paths estruturais em `#F5F2ED`, diagonal em `#FB3640`
- Opcional: glow vermelho muito sutil (`radial-gradient rgba(251,54,64,0.06)`)
  na área do logo formado

### Copy principal

```
font-family: Panchang  weight: 800  font-size: clamp(32px, 5vw, 56px)
letter-spacing: -0.03em  color: #F5F2ED
```

"Boas empresas constroem produtos. Grandes empresas constroem **percepção**."

A palavra em destaque pode receber `color: #FB3640`.

### CTA

Mesmo padrão do Hero CTA primário:
```
background: #FB3640  color: #000F08
font-family: Panchang  weight: 600
```

## 11. Sistema de Destaques

Um destaque é qualquer elemento que precisa de atenção imediata.

A Coded by M tem um único mecanismo de destaque: o vermelho `#FB3640`.

### Hierarquia de destaques

```
Nível 1 — Corte total (maior impacto):
  Fundo vermelho sólido: CTA primário, verso do cartão de visita

Nível 2 — Sinal pontual:
  Elemento vermelho isolado: ponto de hotspot, vértice ativo, conector

Nível 3 — Acento tipográfico:
  Texto vermelho em opacidade 0.7: labels de seção, tags, categorias
  Sublinhado vermelho fino (0.5px): underlines de CTA, separadores

Nível 4 — Presença de ambiente:
  Glow vermelho: radial-gradient opacity 0.06 máximo (hover de cards, CTA Final)
```

Regra: nunca usar dois níveis de destaque no mesmo contexto visual.
O vermelho no fundo do CTA impede uso de glow vermelho no mesmo elemento.

## 12. Sistema de Hover

O hover revela — nunca apenas destaca.

### Fragmentos 3D

```
entrada: scale de 1.0 → 1.06, duration 200ms, ease power2.out
aresta: opacidade de 0.30 → 0.85
ponto ápice: cor de #F5F2ED → #FB3640, escala de 0.6 → 1.2
```

### Botões primários (vermelho)

```
background: #FB3640 → #C42030
transition: background 150ms ease
```

### Botões secundários (borda)

```
border-color: #2a4a32 → #F5F2ED
color: #F5F2ED → #F5F2ED (sem mudança de texto)
transition: border-color 150ms ease
```

### Links de navegação

```
color: #8A8780 → #F5F2ED
transition: color 200ms ease
```

### Cards HTML (ProjectCard, cards de serviço)

```
border-color: #1a2a1e → #2a4a32  (borda ganha temperatura verde)
transition: border-color 200ms ease
```

Nunca usar `box-shadow` colorida em hover de cards. Muda o volume, não a cor.

---

## 13. Sistema de Foco

Foco é acessibilidade e identidade ao mesmo tempo.

```
outline: 2px solid #FB3640
outline-offset: 3px
border-radius: 0  (consistente com a linguagem angular da marca)
```

Não usar `outline: none` em nenhum elemento interativo.

Em contexto escuro, o vermelho `#FB3640` como outline tem contraste suficiente
para WCAG AA sem precisar de cor alternativa.

## 14. Regras de Uso do Vermelho

O vermelho é a cor mais poderosa do sistema. Por isso tem as regras mais rígidas.

**Onde o vermelho DEVE aparecer:**
- CTA primário (fundo ou text)
- Fragmento/hotspot ativo
- Conector entre 3D e HTML overlay
- Diagonal do logo (sempre)
- "by" no wordmark quando em contexto de título
- Labels de seção (cor + opacidade 0.7)
- Linha de `0.5px` sob elementos de destaque tipográfico
- Foco (outline)

**Onde o vermelho PODE aparecer:**
- Hover de links importantes
- Underline de CTA text
- Glow de ambiente (máx `rgba(251,54,64,0.08)`)
- Vértice ativo de fragmento 3D
- Marcador de hotspot (pulsação sutil)

**Onde o vermelho NÃO deve aparecer:**
- Texto de corpo
- Texto secundário ou terciário
- Backgrounds de seção que não sejam CTA Final
- Bordas gerais de layout (usar `#1a2a1e`)
- Sombras de elementos não-interativos
- Geometria 3D que não seja o logo ou o estado ativo
- Gradientes decorativos de página

**Regra de frequência:**
Em qualquer viewport, o olho deve encontrar no máximo 2–3 elementos vermelhos
ao mesmo tempo. Se há mais, algum está errado.

---

## 15. Regras de Uso do Branco Quente

O Off White `#F5F2ED` tem temperatura quente. Isso é deliberado e inviolável.

**Onde o Off White deve aparecer:**
- Todo texto de título (H1, H2, H3, Display)
- Geometria 3D primária (arestas, nós — estado padrão)
- Logo em fundos escuros
- Ícones e símbolos de interface primária

**O que nunca substituir por `#ffffff`:**
- Nenhum texto de título
- Nenhuma aresta ou nó 3D
- Nenhum elemento do logo

**Por quê:**
O branco puro `#ffffff` cria contraste máximo que elimina a temperatura do
ambiente. O `#F5F2ED` mantém o equilíbrio entre legibilidade e sofisticação.

**Gradação do branco:**
Para hierarquia sem usar vermelho, usar a escala de grays quentes:
```
Primário:    #F5F2ED  (título, estrutura 3D principal)
Secundário:  #C8C4BE  (texto de corpo importante)
Terciário:   #8A8780  (subtexto, labels)
Silenciado:  #4A4844  (informação de apoio, timestamps)
```

## 16. Regras de Uso do Verde Profundo

O Forest Dark `#0E1810` e o Deep Black `#000F08` compartilham uma temperatura
verde. Essa temperatura é o que separa a Coded by M de um dark mode genérico.

**Onde o verde aparece:**
- No fundo base `#000F08` — temperatura quase imperceptível
- No Forest Dark `#0E1810` — fundos de cards e overlays
- Nas bordas `#1a2a1e` e `#2a4a32` — separadores com temperatura
- No botão secundário: `border: 1px solid #2a4a32`
- Na barra divisória entre logo e wordmark: `linear-gradient(to bottom, transparent, #2a4a32, transparent)`
- Nas numerações decorativas: `color: #1a2e1e` (Panchang pesado, baixíssima opacidade)

**Regra:**
O verde nunca deve ser visível como cor. Deve ser sentido como temperatura.
Se alguém notar que tem verde ali — está intenso demais.

**O que o verde não deve fazer:**
- Aparecer como cor de texto
- Aparecer como borda visível de alta opacidade
- Criar "tema verde" em nenhuma seção
- Substituir o vermelho como cor de destaque

## 17. Regras para Evitar Aparência Genérica

A principal ameaça à identidade é o visual genérico. Estas regras evitam os
padrões mais comuns que destroem a percepção premium.

**Não usar `#000000` como fundo.** Preto puro é dashboard. Use `#000F08`.

**Não usar `#ffffff` como texto.** Branco puro é bootstrap. Use `#F5F2ED`.

**Não usar azul, roxo ou cyan em nenhum elemento.** Essas cores não existem
no sistema da Coded by M.

**Não usar `border-radius` em elementos estruturais.** Botões, cards e
fragmentos são angulares. Curvas são permitidas apenas em avatares ou elementos
circulares com motivo (hotspot, ponto âncora).

**Não usar gradientes lineares como backgrounds de seção.** Gradientes de
`#000 → #111` são o sinal visual mais genérico de um dark theme sem identidade.
O fundo é `#000F08` uniforme.

**Não usar tipografia sem definir ambos os pares.** Todo texto é Panchang
(display) ou Satoshi (corpo). Nenhum fallback genérico visível ao usuário.

**Não usar o vermelho como cor "temática" de seção.** Só o CTA Final pode ter
uma cena com presença vermelha dominante. Em todas as outras seções, o vermelho
é pontual.

**Não usar animações de "typing cursor" ou contadores genéricos.** A linguagem
da Coded by M é de construção geométrica, não de efeitos tipográficos comuns.

---

## 18. Regras para Manter Aparência Premium

Premium não é excesso de elementos. É precisão em cada um.

**Tipografia bem-espaçada.**
Títulos com `letter-spacing: -0.02em` a `-0.03em` (tight, intencional).
Labels e captions com `letter-spacing: 0.3em` a `0.4em` + `text-transform: uppercase`.
Corpo com `line-height: 1.7`.

**Hierarquia clara de opacidades.**
Não usar mais de 4 níveis de opacidade por tela. Cada nível deve ter propósito
de hierarquia — não de decoração.

**Bordas finas com temperatura.**
`1px solid #1a2a1e` é a borda padrão. `1px solid #2a4a32` é a borda de estado
ativo/hover. Bordas `2px` ou mais espessas só para separadores estruturais de
seção.

**Espaçamento generoso.**
As seções respeitam a escala de espaçamento do Brand Guide: `padding` de
`60px – 100px` no eixo vertical. Espaço vazio é parte do design, não ausência
de conteúdo.

**Elementos de apoio com baixíssima opacidade.**
Numerações decorativas (`01`, `02`) em Panchang pesado com `color: #1a2e1e` —
a cor verde tão escura que mal aparece. Esse detalhe é a diferença entre
template e identidade.

**Linhas de `0.5px`.**
Separadores e underlines de destaque tipográfico usam `0.5px` de espessura.
Isso é uma decisão premium — bordas grossas são rudimentares.

## 19. Relação Entre a Identidade Visual e o Conceito

**"Nada surge pronto. Tudo é construído."**

Este conceito não é apenas narrativo. Ele é estrutural. E a identidade visual
o expressa em cada camada:

**O logo é uma construção.**
Não é um símbolo estático — são linhas que poderiam ser desenhadas na sequência
exata em que a animação do Loading as revela: estrutura esquerda, estrutura
direita, diagonal vermelha.

A animação do Loading não imita o logo. O logo foi desenhado para poder ser
animado como construção.

**O Off White é o material.**
As estruturas — nós, arestas, terreno — são da mesma cor que o texto. Isso diz:
a informação e a estrutura são feitas do mesmo material. Código e design nascem
juntos.

**O vermelho é o corte que revela.**
A diagonal do logo corta as duas estruturas. No site, o vermelho sempre marca
o momento de revelação: o projeto foi descoberto (hotspot), a ação está clara
(CTA), a narrativa chegou ao fim (CTA Final).

**O Forest Dark é o espaço de trabalho.**
Cards e overlays em `#0E1810` parecem estruturas dentro do ambiente escuro —
não caixas brancas sobrepostas. O trabalho está acontecendo dentro do espaço,
não sobre ele.

**A névoa é o horizonte desconhecido.**
O fog do Three.js em `#000F08` sugere que a paisagem continua além do que o
olho alcança. O projeto é maior do que o que está visível. Há mais para descobrir.

Cada decisão visual reforça o mesmo princípio: algo está sendo construído,
com precisão, diante de quem olha.

## 20. Como Aplicar Esta Direção Visual aos Experimentos Já Existentes

Esta seção descreve as mudanças visuais necessárias em cada experimento para
alinhá-los à identidade oficial da Coded by M.

Nenhuma mudança de lógica, arquitetura ou comportamento é necessária.
Apenas configuração visual.

---

### 20.1 Triangle Loader

**Estado atual:** fundo `#050505`, pontos e linhas em branco neutro/cinza.

**Mudanças necessárias:**

`config.ts` — COLORS:
```
before: background: '#050505'
after:  background: '#000F08'

before: point: '#ffffff' (ou similar)
after:  point: '#F5F2ED'

before: line: '#888888' ou '#aaaaaa'
after:  line: '#F5F2ED'  lineOpacity: 0.75
```

O ponto central (onde as três arestas se encontram) pode receber `#FB3640`
quando o triângulo está completo — o "sinal" de que a estrutura foi formada.

Ou, para máxima fidelidade ao logo: ao completar a construção, uma quarta linha
(a diagonal vermelha do logo) pode cortar o triângulo — transformando o Loading
em uma prévia do símbolo completo.

### 20.3 Terrain Mesh

**Estado atual:** fundo `#050505`, terreno em wireframe neutro.

**Mudanças necessárias:**

`config.ts` — COLORS:
```
background: '#000F08'
```

FOG:
```
color: '#000F08'  (já pode estar correto, verificar)
```

LAYERS — opacidades e cores por camada:

### 20.4 Project Fragments

**Estado atual:** fragmentos em wireframe neutro, sem diferenciação visual de
estado ativo.

**Mudanças necessárias:**

`config.ts`:
```
fragment.defaultColor:  '#F5F2ED'  opacidade: 0.30
fragment.activeColor:   '#F5F2ED'  opacidade: 0.85
fragment.apexColor:     '#FB3640'  (vértice superior — o sinal vermelho)
```

Marcador de hotspot:
```
cor: '#FB3640'
pulsação: amplitude 0.12, período 2.5s (já implementada na `useDiscovery` ou
adicionar em `useFragmentBuild`)
```

Cursor: ao hover, trocar para `pointer` (já implementado).

O fragmento descoberto deve ter o vértice ápice vermelho visível mesmo antes do
hover — o hotspot que chama atenção. Ao hover, esse ponto cresce (`escala 0.4 → 1.2`).

### 20.6 Scroll Camera

**Estado atual:** herda as configurações visuais de TerrainMesh, ProjectFragments
e HtmlOverlay. A UI de "Scroll ↓" usa tipografia neutra genérica.

**Mudanças necessárias:**

As mudanças dos experimentos anteriores (TerrainMesh, ProjectFragments,
HtmlOverlay) propagam automaticamente para o ScrollCamera — ele os reutiliza.

Dica de scroll (hint UI):
```
"Scroll":  font Satoshi  size: 10px  tracking: 0.4em  uppercase  color: #4A4844
"↓":       color: #4A4844
```

Ao iniciar o scroll, o hint faz `opacity: 0` com `transition: opacity 500ms ease`.

Não alterar lógica, keyframes ou comportamento de câmera — apenas visual.
