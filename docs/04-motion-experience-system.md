# 04 — Motion & Experience System

# Coded by M — Sistema de Movimento e Experiência

## Objetivo

Definir a linguagem de movimento da Coded by M.

Este documento não descreve componentes específicos do site.

Ele define como a marca se comporta visualmente.

Toda animação futura deve respeitar estes princípios.

---

# Filosofia

A Coded by M é um estúdio de Web Design Premium.

O movimento não existe para impressionar.

O movimento existe para reforçar a narrativa.

Cada animação deve contribuir para a percepção de:

- Precisão
- Elegância
- Performance
- Detalhismo

Se uma animação não contribui para esses valores, ela não deve existir.

---

# Princípio 01 — Construção Antes da Forma

Nada surge pronto.

Tudo é construído.

Linhas conectam pontos.

Formas emergem de estruturas.

Estruturas se transformam em experiências.

Esta é a regra mais importante de toda a linguagem de movimento da marca.

---

## Aplicação

### Errado

Um triângulo aparece instantaneamente.

### Correto

Pontos surgem.

Linhas conectam os pontos.

A estrutura é construída.

O triângulo aparece como consequência.

---

# Princípio 02 — Movimento com Propósito

Toda animação deve comunicar algo.

Jamais utilizar movimento apenas para parecer moderno.

Movimento deve indicar:

- construção
- conexão
- descoberta
- transformação
- evolução
- organização
- refinamento

Nunca distração.

---

# Princípio 03 — Elegância Sobre Velocidade

Movimentos devem ser:

- suaves
- controlados
- precisos
- cinematográficos
- bem temporizados

Evitar:

- movimentos agressivos
- acelerações exageradas
- overshoots excessivos
- efeitos chamativos sem contexto
- excesso de bounce
- excesso de neon
- comportamento de videogame

Referência emocional:

Porsche, não videogame.

---

# Princípio 04 — Estruturas Vivas

As estruturas da Coded by M nunca são completamente estáticas.

Mesmo em repouso devem transmitir vida.

Exemplos:

- partículas lentas
- respiração da malha
- leve deslocamento de profundidade
- iluminação dinâmica discreta
- rotação sutil
- variação mínima de opacidade

Tudo deve ser de baixa intensidade.

---

# Princípio 05 — Profundidade Cinematográfica

O usuário não navega páginas.

O usuário percorre ambientes.

A sensação desejada é:

“Estou atravessando uma estrutura digital.”

Sempre que possível trabalhar com:

- foreground
- midground
- background
- parallax controlado
- câmera lenta
- camadas visuais
- objetos em profundidade

Criar sensação de ambiente, não apenas de layout.

---

# Princípio 06 — A Experiência Começa Antes da Home

O loading não é uma tela separada.

O loading é o primeiro capítulo da narrativa.

O site não começa depois do carregamento.

O site começa no carregamento.

---

# Sistema de Formas

## Forma Principal

Triângulo.

Representa:

- estrutura
- estabilidade
- direção
- crescimento
- precisão

O triângulo é a unidade simbólica principal da Coded by M.

---

## Forma Secundária

Malhas trianguladas.

Representam:

- sistemas
- construção
- evolução
- organização
- conexão

---

## Forma Terciária

Terrenos geométricos.

Representam:

- jornada
- exploração
- portfólio
- projetos
- evolução

---

## Partículas Conectadas

Representam:

- ideias
- conexões
- processo criativo
- início de uma estrutura

---

# Sistema de Movimento

## Tipo 01 — Construção

Uso:

- Loading
- Hero
- CTA Final
- Entradas importantes

Comportamento:

- pontos surgem
- linhas conectam
- formas são desenhadas
- estrutura nasce

---

## Tipo 02 — Expansão

Uso:

- transições
- mudança de cenas
- crescimento de malhas

Comportamento:

- uma estrutura cresce
- novos elementos aparecem
- a cena ganha profundidade

---

## Tipo 03 — Conexão

Uso:

- hover
- interações
- hotspots
- projeto selecionado

Comportamento:

- linhas unem elementos
- fragmentos reagem
- informações emergem

---

## Tipo 04 — Transformação

Uso:

- momentos UAU
- CTA Final
- transições principais

Comportamento:

- estruturas assumem novas formas
- fragmentos convergem
- malhas se reorganizam

---

# Sistema de Câmera

## Velocidade

Lenta.

Controlada.

Cinematográfica.

## Sensação

Observação.

Nunca perseguição.

Nunca corrida.

Nunca frenética.

## Objetivo

Permitir contemplação.

Permitir descoberta.

Criar presença.

---

# Sistema de Scroll

O scroll é narrativo.

O usuário deve sentir que está avançando na história.

Não deve parecer apenas uma sequência de seções.

Cada scroll deve revelar algo novo.

Scroll deve controlar:

- construção de estruturas
- entrada de textos
- avanço de câmera
- transformação de malhas
- revelação de projetos
- convergência final

---

# Sistema de Hover

Hover não deve apenas destacar.

Hover deve revelar.

Exemplos:

- estruturas se conectam
- linhas surgem
- fragmentos respondem
- informações emergem
- bordas são desenhadas
- o terreno reage ao cursor

---

# Sistema de Projetos

Os projetos não aparecem em uma grade convencional na Home.

Os projetos fazem parte de uma paisagem digital.

Cada projeto representa um fragmento da estrutura da Coded by M.

Ao explorar a paisagem:

- projetos emergem
- histórias são reveladas
- conexões são percebidas
- cards HTML aparecem como overlays

O usuário descobre os projetos.

Não apenas os visualiza.

---

# Sistema de Iluminação

Ambiente predominante:

Estúdio Escuro.

Características:

- preto profundo
- contraste elevado
- iluminação pontual
- profundidade
- luz lateral
- brilho discreto em linhas
- ausência de neon genérico

Evitar:

- excesso de glow
- excesso de brilho
- aparência cyberpunk comum
- visual genérico preto + azul neon

---

# Sistema de Loading

## Conceito

Um triângulo pequeno é construído por linhas e pontos.

Depois de construído, gira lentamente.

O loading deve mascarar o carregamento do restante do site.

Mas também deve introduzir o princípio central:

Nada surge pronto. Tudo é construído.

## Comportamento

1. Tela escura
2. Ponto 1 aparece
3. Ponto 2 aparece
4. Ponto 3 aparece
5. Linhas conectam os pontos
6. Triângulo wireframe se forma
7. Triângulo gira lentamente
8. Estrutura ganha profundidade
9. Câmera aproxima
10. Transição para Intro

---

# Regra Final

A experiência deve fazer o visitante pensar:

“Esse cara faz sites absurdamente bem feitos.”

E não apenas:

“Esse site tem animações legais.”

A experiência existe para valorizar o trabalho.

Nunca para competir com ele.
